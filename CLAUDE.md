# 面接無双 — CLAUDE.md

## プロジェクト概要
就活生向けの面接練習Webアプリ「面接無双」。日本人面接官との対話形式で一次〜最終面接を練習でき、回答に対してルールベースの5観点添削とコーチチャットでフィードバックを返す。パスワードゲート付きで友人限定に公開中。

## 技術スタック
- **単一HTMLファイル**の静的サイト（`index.html`）。ビルド不要・外部通信なし・フレームワーク不使用（素のJS/CSS）
- 画像はすべてbase64埋め込み（面接官の写真6枚、ヒーロー写真等）
- データ永続化は`localStorage`のみ（`mm_gate_v1`＝パスワードゲート通過フラグ、`mensetsu_history_v1`＝面接履歴、最大50セッション）
- ホスティング: GitHub Pages（リポジトリ直下の`index.html`をそのまま配信）

## ディレクトリ構成の要点
```
mensetsu-musou/
├── index.html   # アプリ本体（このファイルが全て。単一ファイルで完結）
├── README.md    # ユーザー向け概要・クレジット
└── CLAUDE.md    # このファイル
```
- 実際の編集作業は基本的にセッションのscratchpad（`mensetsu-musou.html`）で行い、Pythonスクリプト（`edit_v13.py`, `edit_v14.py`...という命名）でanchor置換方式で編集してから、この`index.html`へ`cp`する運用
- リサーチノート（企業別質問データの出典等）もscratchpad側にあり、このリポジトリには含めていない

## 現在の進捗（v14編集で対応中）
評価レポートの質を上げる改修に着手中。対応項目:

**本スコープ（優先度高）**
1. スコア説明力バグ修正 — `analyze()`のscoreが全findingsから計算される一方、画面表示は減点理由を最大3件に切り詰めていて不一致だった。重み付け（構造的欠陥=1点／文体レベルの軽微な癖=0.5点）＋「ほか+N件」表示で解消
2. `DEEP_TYPES`拡張 — 数字・構成の評価軸が`motivation`/`industry`タイプの質問に適用されていなかった問題を修正
3. 5観点メーターの小標本(n)配慮 — 対象問題数が少ない軸に「参考」ラベルを表示し、%の過大解釈を防止
4. 自由質問欄のEnterキー問題修正 — `<input type="text">`だったため改行できず即送信されていたのを`<textarea>`化。Shift+Enterで改行、Enterで送信
5. `routeFreeText`/コーチ返答の分岐拡張 — ルールベースのまま「短く」「他社」「自己紹介」「緊張」等の分岐を追加

**軽微2項目（本スコープに含めて一緒に対応）**
- コーチチップ（なぜこの評価？等）に既読の見た目をつけて連打時の重複感を軽減
- 総評のダメ出しフォールバックを配列順ではなく重要度順（`BAD_PRIORITY`）でソート

現在の状態: `edit_v14.py`の実装・構文チェック・Node上でのスコアリングロジックのスモークテストまで完了。ブラウザでのプレビュー検証はこれから。

## 直近の決定事項
- **本物のClaude API連携（自由質問への真の対話応答）は別タスクとして切り出す**。理由: このアプリは静的HTML＋GitHub Pagesでバックエンドがなく、APIキーをHTMLに埋め込むとソースから誰でも読めて漏洩する。実現するには製品版でキーを守るバックエンド（プロキシ）が必要になるため、今回のスコープからは意図的に外している
- `company_specific`判定のゲーム化しやすさ（社名を1回入れるだけで加点される問題）は今回は優先度低として保留
- 作業は5時間の利用上限を考慮し、可能な限り細かい単位でgitにコミット/pushして進捗を残す方針

## 実行・テストコマンド
このプロジェクトにはビルド・パッケージマネージャは無い。動作確認は以下の手順。

```bash
# ローカルプレビュー（任意のポートで配信するだけ）
python3 -m http.server 8642 --directory .
# → http://localhost:8642/index.html をブラウザで開く

# JS構文チェック（<script>内を抜き出してNodeでチェック）
python3 -c "
import re
h = open('index.html', encoding='utf-8').read()
scripts = re.findall(r'<script>(.*?)</script>', h, re.S)
open('/tmp/_main_script.js','w',encoding='utf-8').write(max(scripts, key=len))
"
node --check /tmp/_main_script.js
```

デプロイは`git add index.html && git commit && git push`のみ（GitHub Pagesが自動反映、1〜2分）。
